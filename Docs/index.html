Annotations

Every annotation begin with a @ and must be alone in the line wich it is present. Separated from the annotation declaration by one single blank space comes a set of parameters separated of each other by one single blank space. The following are the currently supported annotations: @file_path, @file_name, @file_ext, @file_langs, @features
Annotations should match the following regular expression: @[a-zA-Z0-9]+\n.*

@file_path

Required at the top of the file. One parameter. The @file_path annotation defines the path the output file will be written inside the Generateds folder. The not pre-existent folders are goind to be created inside of the product folder.

@file_name

Required at the top of the file. One parameter. The @file_name annotation defines the name of the output file

@file_ext

Required at the top of the file. One parameter. The @file_ext annotation defines the file extension of the output file

@file_langs

Required at the top of the file. One or more parameters. The @file_langs annotation defines the set of languages that are going to be present in the file. E.g. A php page may contain html, php and javascript

@features

Used on the top of a statement. One or more parameters. The @features annotation defines the set of products that implements that piece of code
